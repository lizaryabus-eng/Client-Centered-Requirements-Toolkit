 Client-Centered Requirements Toolkit  
Полный комплект документации учебного проекта, посвящённого сбору, анализу и формализации требований с точки зрения клиента.  
Документация оформлена как реальный аналитический проект: включает интервью, анализ проблем, User Stories, Use Cases, Value Proposition, а также шаблоны.

Содержание
- Понимание проблем клиента  
- Формирование требований  
- Подготовка артефактов для разработчиков  
- Примеры аналитической документации  
- Полностью структурированный репозиторий  

Цель проекта  
Показать, каким образом требования клиента проходят путь:  
Проблема → Интервью → Анализ → User Stories → Use Cases → Требования продукта.**

Как использовать
Скачайте репозиторий и изучайте документацию по шагам — она уже полностью заполнена и готова к использованию в учебных работах.

Структура
- docs — вся основная документация  
- templates — шаблоны требований
